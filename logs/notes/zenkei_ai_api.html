      • data_generator
        • batch_size
          • the size of a batch
        • file_list
          • a list of filenames of images
        • label_list
          • a list of the label corresponding to file_list
        • shuffle=False
          • shuffle the sequence of data
        • is_gray_image=False
          • the image data are formed as 1 channel array
        • bgr=False
          • set True if the order of image channel should be BGR otherwise it is RGB
        • fixed_norm=None
          • If None: normalize the pixel value for each image
          • otherwise, use 'fixed_norm[0]' for the mean, fixed_norm[1] for the standard deviation
        • no_std_norm=False
          • If fixed_norm is not None, this has no effect
          • otherwise, if set True, pixels are not divided by their standard deviation
            • True: [-128, 128]
            • False: [-1, 1]
        • min_max_norm=False
          • If set True, fixed_norm and no_std_norm are ignored
          • the pixel values are normalized to [0,1] by the minimum and the maximum
        • ouput_format='numpy'
          • 'numpy' or 'torch'

    • torch_utils
      • accuracy(outputs, labels)
        • calculate accuracy from model "outputs" and target "labels"
        • input: outputs, labels
      • seq2seq_reg(ouput, xtra, loss, alpha, beta)
      • plot_lrf(lrf_data, ylim, zlim, ax, with_points, with_errorbars)
        • plot the result of the learning rate finder
        • inputs
          • lrf_data: list/tuple or str - the result of LR finder
      • print_model_size(model)
        • print model size information (num_trainable_parameter, num_non_trainable_parameter, total_num_parameter)
      • plot_history(history)
        • plot training history of a model includes accuracy and loss
      • plot_histories(hs, labels, scale_fixed, xrange, n_ave, logscale)
        • plot multiple history
      • Class: model
        • init
          • model: torch.nn.Module
          • device: torch.device
          • opt_func: torch.optim
          • loss_dict: dictionary of the label (key) and the loss or metric function
            • eg: {'loss': F.cross_entropy, 'acc': accuracy}
            • 'loss' entry is mandatory
          • layer_groups
            • None or list/tuple
            • layer groups for which different LRs and/or momentums can be set
          • is_rnn: bool - set 'True' for RNN model which outputs extra elements
          • reg_seq2seq - only for RNN models to use regularization 'req_seq2seq'
        • freeze_to(n)
          • freeze to the n-th layer of the model
          • input: n
        • unfreeze()
          • unfreeze the whole model
        • learning rate
          • find_lr
            • trn_iter: data loader providing input data as well as the labels
            • ave_steps: Int - number of run to take ensemle average of the results
              • if 0 is given, losses are evaluated in a continuous mode where the weight are not reset for each lr
              • default is 1, no ensemble averaging
            • lr_steps: Int - steps to optimize for each lr value
            • lr_factor: Float - the multiplicative factor for the learning rate for the next cycle
              • default value is 1.1
            • max_lr: Float - the maximum lr value to search
              • if None is given, searching is determined by the loss limit which is 4 times bigger than the initial loss value
              • default: None
            • lrs: Float or list/tuple/np.array of Float - the initial value of the learning rates for LR finder
              • if list/tuple/np.array of Float is given, the length must be equal to the number of layer groups
              • default value: 1.0e-6
            • wds: Float or list/tuple/np.array of Float - weight decay factor
              • default value: 1.0e-7
            • wd_type: Int
              • '0' is given - AdamW type weight decay is used
              • otherwise - conventional L2 regularization type is used
            • clip: None or Float - gradient clip value
            • grad_acc: Int - gradient accumulation factor
              • eg: '2' is given, updating parameters are applied once in 2 steps -> batch size become twice
              • default: 1
            • verbose: Int

        • fit
          • trn_iter
            • Training data loader
          • val_iter=None
            • Validation data loader
          • epochs=1
            • Number of epoch for training
          • lrs=1e-3
            • Learning rate for training,
            • If use_clr=True, this is the maximum (peak) value for the cycle
          • wds=1e-7
            • Weight decay factor
          • wd_type=0
            • Weight decay type
          • clip=1
            • Gradient clipping value
          • use_clr=True
            • Cyclical learning rate
          • clr_lr_factor=10
            • ratio of peak LR value relative to the starting LR value
          • clr_peak=0.25
            • Position of the LR peak relative to the entire training epochs
            • 0.25 means 1/4
          • clr_final=0.075
            • Duration of the final annealing relative to the entire training epochs
          • clr_moms=(0.95, 0.85)
            • Values of optimizer's momentum at the beginning and at the peak of LR given by cl_peak
          • grad_acc=1
            • Gradient accumulation factor
          • feacher_forcing=None
            • This is only for RNN mdoels.
            • Number of epochs for which teacher forcing is used.
          • verbose=0
            • 0: prints nothing
            • 1: print current loss at each epoch
            • 2: also prints progress bars
            • 3: for google colaboratory
        • evaluate

            • Validation data loader
          • eval_func
            • Function like eval_func(ouputs, labels) which returns preds and labels
          • verbose=0
